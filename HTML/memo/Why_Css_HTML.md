# 왜 HTML과 CSS를 배워야 할까?

게시판이나 블로그를 좀더 자유롭게 꾸밀 수 있습니다.
```
    웹을 이용하다 보면 알게 모르게 HTML을 자주 만납니다.
    다양한 웹사이트, 인터넷 카페의 게시판, 메일 , 블로드 등은 모두 HTML과 CSS를 사용하고 있는 곳입니다.
    HTML을 익히면 게시판에 글자나 이미지등을 자유롭게 올릴 수 있습니다.
    또 블로그에서 HTML과 CSS를 사용해서 원하는 형태의 레이아웃과 스킨을 만들 수 있습니다.

```

<br/>

웹 사이트 제작을 위해서는 HTML과 CSS가 필수입니다.
```
    웹 사이트를 제작할 때도 HTML과 CSS는 필수 입니다. 많은사림이 우베사이틀르 제작할 때 드림위버 같은 프로그램이나 워드프레서, 제로보드 같은 공개 소스를 이용합니다. 
    드림위버가 HTML이나 CSS 소스를 자동으로 만들어 주긴 하지만. 정작 스 소스를 이해하지 못한다면 원하는 대로 사이트 만들기가 쉽지 않습니다.
    워드프레스나 제로보드 같은 공개소스를 원본 그래도 사용해도 사이트는 만둘 수 있지만. 필요에 따라 내 취향에 맞게 사이틀르 바꾸고 싶거난 좀 더 전문적인 사이트로 바꾸고 싶을 경우.
    HTML울 알지 못한다면 어디를 어떻게 수정해 하는지 손을 댈 수 없습니다. 틀히 이런 공개 소스의 핵심이 되는 스킨을 우리가 만든 사이트에 적용하여면 어떤 소스를 건드려야 내가 원하는 부분이 바뀌는지 제대로 알고 있어야 합니다.
    이와 같이 어떤 전문 프로그램을 이용해서 웹사이틀르 만들 경우. 여러 소스를 적용하려면 가장 기본이  되는 HTML과 CSS를 제대로 이해하고 다룰 수 있어야 합니다.
```

# 웹 표준은 무엇인가?

왜 웹 표준이 필요할까요?
```
    웹표준은 왜 지켜야 할까? 어떻게든 여러 기술을 사용해서 남들에게 멋있게 보여주기만 하면 되는 것 아닐까요??
    예를 들어. 옷 한 벌을 구힙한다고 가정해 보겠습니다. 매장에 있는 옷들은 모두 표준화된 치수를 사용하여 만들어집니다.
    따라서 유명 백화점이든 시장이든 장소와 상관없이 자신의 치수만 알면 손수베게 옷을 구입할 수 있습니다.
    이런 표준 신체 치수가 없다면 가는 곳마다 일일이 옷을 입어보고 사야할 것 입니다.
    HTML도 마찬기지 ㅇ비니다. 어떤 사이트는 인터넷 익스플로러에서만 구현되고, 또 어떤 사이트는 크롬에만 구현된다고 하면 사용자들이 자유롭게 사이트를 이용할 수 없습니다.
    각각의 브라우저에 맞게 사이트를 다시 만들어애 한다면 정말 비효율 적 일 것입니다. 더구나 요즘은 인터넷에 접속할 수 있는 기기들이 더 많아져.
    PC뿐 아니라 스마트폰, 테블릿, TV까지도 인터넷이 연걸됩니다. 만약 우리 회사 홈페이지를 PC에서 보여줄 때 따로 만들고, 아이폰용이나 안드로이드폰용등 기기별로 띠로 만들어야 한다면 사회적으로 엄천한 시간과 비용이 낭비될 것입니다.
    웹 표준은 이런 비효율 적인 것들을 줄여 줍니다. 웹 표준을 지켜 사이트를 제작하면 일반 사용자는 장소와 기기에 상관없이 쉽게 볼 수 있고, 웹 개발자와 디자이너들은 시간을 절약할 수 있겠죠.
    웹 표준으로 문서 하나를 만들면 어떤 기기에서도 볼 수 있기 때문입니다.
    이렇듯 앞으로는 HTML5로 문서를 만드는 것이 웹 표준을 지킨 문서를 만드는 데 필수적이 될 것 입니다.  

```
